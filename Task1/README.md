# Анализ проблемных мест системы "Будущее 2.0"

## Выявленные проблемы

### Технические проблемы

| ID | Проблема | Описание | Последствия для бизнеса |
|----|----------|----------|-------------------------|
| P01 | Устаревшая версия СУБД | SQL Server 2008 не поддерживается вендором с 2019 года | Риски безопасности, невозможность использовать современные возможности БД, проблемы с производительностью |
| P02 | Монолитная архитектура DWH | Вся бизнес-логика сосредоточена в одном месте | Невозможность независимого развития направлений бизнеса, долгий time-to-market |
| P03 | Отсутствие масштабируемости | Система не может горизонтально масштабироваться | Часовые ожидания отчетов, невозможность обработки растущих объемов данных |
| P04 | Tight coupling между системами | Сильная связанность компонентов через DWH | Изменение в одной части требует изменений во всей системе |
| P05 | Устаревшая интеграционная шина | Apache Camel как единая точка интеграции | Bottleneck в интеграциях, сложность добавления новых систем |
| P06 | Отсутствие self-service BI | Нет инструментов для самостоятельного построения отчетов | Бизнес-пользователи зависят от ИТ, долгое получение аналитики |
| P07 | Смешение OLTP и OLAP нагрузок | Операционные и аналитические запросы в одной БД | Взаимное влияние на производительность, деградация скорости работы |
| P08 | Устаревший UI (Power Builder) | Технология 90-х годов для клиентского интерфейса | Сложность найма разработчиков, плохой UX, невозможность мобильного доступа |

### Архитектурные проблемы

| ID | Проблема | Описание | Последствия для бизнеса |
|----|----------|----------|-------------------------|
| P09 | Отсутствие доменной модели | Нет четкого разделения на бизнес-домены | Сложность понимания системы, дублирование логики |
| P10 | Централизованное управление данными | Все данные управляются через единый DWH | Bottleneck в развитии, команды не могут независимо работать с данными |
| P11 | Отсутствие событийной архитектуры | Синхронные вызовы между компонентами | Высокая связанность, сложность добавления новой логики |
| P12 | Нет разделения ответственности | DWH отвечает за всё: хранение, трансформацию, бизнес-логику | Сложность поддержки и развития |

### Безопасность и комплаенс

| ID | Проблема | Описание | Последствия для бизнеса |
|----|----------|----------|-------------------------|
| P13 | Смешение медицинских и финансовых данных | Нет изоляции между критичными типами данных | Риски нарушения регуляторных требований (ФЗ-152, PCI DSS) |
| P14 | Отсутствие централизованной системы доступа | Управление доступом на уровне DWH | Сложность аудита, риски несанкционированного доступа |
| P15 | Нет системы аудита | Отсутствует централизованный аудит действий | Невозможность расследования инцидентов, проблемы с регуляторами |

### Организационные проблемы

| ID | Проблема | Описание | Последствия для бизнеса |
|----|----------|----------|-------------------------|
| P16 | Зависимость от легаси-компетенций | Power Builder, старые версии SQL Server | Риск потери экспертизы, высокая стоимость поддержки |
| P17 | Отсутствие DevOps практик | Ручные деплои, нет CI/CD | Долгий цикл поставки, риски при релизах |
| P18 | Нет data governance | Отсутствуют политики управления данными | Дублирование данных, несогласованность, проблемы с качеством |
